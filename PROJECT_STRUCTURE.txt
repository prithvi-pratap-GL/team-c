   - QdrantClientManager class (singleton)
   - Methods: __init__, get_client(), create_collection(), health_check()

   - Vector size: 768
   - Distance: COSINE
   - Status: READY

   - PDFParser class
   - Method: parse(file_path) -> dict
   - Extracts: text, page_count, filename
   - Status: READY

3. txt_parser.py (TASK 2)
   - TxtParser class
   - Method: parse(file_path) -> dict
   - Extracts: text, filename
   - Encoding: UTF-8
   - Status: READY

4. metadata_processor.py (TASK 3)
   - MetadataProcessor class
   - Method: validate_and_process(metadata) -> dict
   - Required fields: department, category, version
   - Generates: doc_id (UUID)
   - Status: READY

5. fixed_chunker.py (TASK 4)
   - FixedChunker class
   - Size: 512

   - Method: chunk(text) -> list
   - Status: READY

6. advanced_chunker.py (TASK 4)
   - AdvancedChunker class
   - Separators: ["\n\n", "\n", ". ", " "]

   - Overlap: 64

   - Status: READY

PAYLOAD SCHEMA (FROZEN)
======================

Every Qdrant point contains:
{
    "chunk_id": str,              # UUID
    "doc_id": str,                # UUID
    "doc_name": str,              # filename
    "department": str,            # normalized
    "category": str,              # normalized
    "version": str,               # normalized
    "doc_date": str | null,       # ISO format
    "document_type": str,         # pdf or txt
    "chunk_text": str,            # content
    "chunking_strategy": str      # fixed or advanced
}

DEPENDENCIES & DESIGN
====================

Singleton Pattern: Embedder, QdrantClientManager
  Reason: Expensive initialization, single source of truth

RecursiveCharacterTextSplitter: Both chunkers
  Reason: Stable, deterministic, structure-aware

BAAI/bge-base-en-v1.5: Embedding model
  Reason: Free, high-quality, local inference, 768 dimensions

Dual Chunking Strategies: Fixed + Advanced
  Reason: A/B testing retrieval quality (Person 4's requirement)

Frozen Payload Schema: All 10 fields
  Reason: Unblock Person 4 immediately without code changes
